**日期:** 2026-05-09
**版本:** v1.0
**状态:** 已实施

## 1. 设计目标

**核心原则: 开发者友好**

- 不因评分阻止技能上架
- 提供清晰的改进建议
- 激励高质量技能

---

## 2. 评分维度

### 2.1 安全评分 (Security) - 阻塞式

| 级别 | 分值 | 行为 |
|------|------|------|
| CRITICAL | 0 | 阻止上架，需修复 |
| HIGH | 30 | 扣分，建议修复 |
| MEDIUM | 70 | 扣分，可选修复 |
| LOW | 90 | 轻微提示 |

**检测规则 (60+ 模式):**

- 命令注入: eval(), exec(), os.system()
- 硬编码密钥: password, api_key, secret
- 恶意下载: wget, curl 下载可执行文件
- SQL/NoSQL 注入: 字符串拼接
- SSRF: localhost 绕过
- 广告追踪: googleads, facebook pixel

### 2.2 独立性评分 (Independence) - 加分制

**原则:** 无阻塞，只影响评分等级

| 加分项 | 分数 |
|--------|------|
| 纯本地计算 | +20 |
| 使用环境变量 | +15 |
| 有超时设置 | +10 |
| 有错误处理 | +10 |
| 使用临时文件 | +5 |

**等级:**

| 等级 | 分数范围 | 标签 |
|------|----------|------|
| A | 90-100 | 高度独立 |
| B | 70-89 | 较独立 |
| C | 50-69 | 有外部依赖 |
| D | 30-49 | 强外部依赖 |
| F | 0-29 | 高度依赖 |

**警告项 (不影响分数):**

- 动态包安装
- 无超时设置
- 下载可执行文件

## 3. 审核流程

```
Skill 上传
    ↓
┌─────────────────────────────────────────┐
│         audit-agent.js                    │
│                                         │
│  ┌───────────────┐  ┌───────────────┐  │
│  │  Python 脚本  │  │ Independence  │  │
│  │  (安全审核)   │  │   Scorer     │  │
│  │  Tier 1-4    │  │  (独立性)    │  │
│  └───────┬───────┘  └───────┬───────┘  │
│          │                  │          │
│          └────────┬─────────┘          │
│                   ↓                   │
│          ┌─────────────────┐             │
│          │  综合审核结果  │             │
│          └─────────────────┘             │
└─────────────────────────────────────────┘
                   ↓
            审核结果 + 评分 + 建议
```
