  <name>Task 3: Add overflow protection to claimRecoverableReputation()</name>
  <read_first>
    - contracts/StakingManager.sol (lines 156-190)
  </read_first>
  <action>
    The `maxRecovery` calculation in claimRecoverableReputation() can overflow:

    ```solidity
    // Current (vulnerable):
    uint256 maxRecovery = (originalSlashAmount[msg.sender] * RECOVERY_RATE_PER_MONTH * monthsElapsed) / 10000;
    ```

    Fix using SafeMath or checked arithmetic:

    ```solidity
    // Option A: Using OpenZeppelin SafeMath
    import "@openzeppelin/contracts/utils/math/SafeMath.sol";
    
    // In function:
    uint256 temp = originalSlashAmount[msg.sender].mul(RECOVERY_RATE_PER_MONTH);
    temp = temp.mul(monthsElapsed);
    uint256 maxRecovery = temp.div(10000);

    // Option B: Using Solidity 0.8+ checked math (recommended)
    // Solidity 0.8+ reverts on overflow by default, but add explicit checks:
    require(monthsElapsed <= 120, "Max 10 years"); // Cap at 10 years
    uint256 maxRecovery = originalSlashAmount[msg.sender] * RECOVERY_RATE_PER_MONTH * monthsElapsed / 10000;
    ```

    Choose Option B (Solidity 0.8+ native overflow protection) as contracts already use ^0.8.20.
  </action>
  <acceptance_criteria>
    - grep -q "monthsElapsed <= 120" contracts/StakingManager.sol OR
    - grep -q "SafeMath" contracts/StakingManager.sol
    - npx hardhat compile succeeds
  </acceptance_criteria>
</task>

  <name>Task 4: Apply CEI pattern to slashLiker()</name>
  <read_first>
    - contracts/StakingManager.sol (lines 93-120)
  </read_first>
  <action>
    Apply Checks-Effects-Interactions pattern to slashLiker():

    Current order issues:
    1. Updates state (lockedAmount) BEFORE external call
    2. External call (hasLiked[_liker] = false) after state update

    Fix to CEI pattern:
    ```solidity
    function slashLiker(address _liker, int256 _penalty, string memory _reason) external onlyOwner {
        // CHECKS
        require(_liker != address(0), "Invalid address");
        require(_penalty < 0, "Penalty must be negative");

        // EFFECTS (state updates before external calls)
        uint256 penaltyAmount = uint256(-_penalty);
        ReputationLock storage lock = reputationLocks[_liker];
        lock.lockedAmount += penaltyAmount;
        if (lock.lastClaimTime == 0) {
            lock.lastClaimTime = block.timestamp;
        }
        originalSlashAmount[_liker] += penaltyAmount;
        userReputation[_liker] += _penalty;
        
        emit ReputationLocked(_liker, penaltyAmount);
        emit AntiSlash(_liker, _penalty, _reason);

        // INTERACTIONS (external calls last)
        hasLiked[_liker] = false;
    }
    ```
  </action>
  <acceptance_criteria>
    - Event emissions appear before external state modifications
    - npx hardhat compile succeeds
    - Tests pass
  </acceptance_criteria>
</task>
