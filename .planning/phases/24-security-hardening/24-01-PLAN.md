  <name>Task 3: Fix overflow in claimRecoverableReputation()</name>
  <files>contracts/StakingManager.sol</files>
  <action>
Fix potential integer overflow in the recovery calculation at line 174.

Current vulnerable code:
```solidity
uint256 maxRecovery = (originalSlashAmount[msg.sender] * RECOVERY_RATE_PER_MONTH * monthsElapsed) / 10000;
```

Change to safe calculation using SafeMath pattern (Solidity 0.8.x built-in):
```solidity
// Calculate recovery with overflow protection
// Use checked arithmetic to prevent overflow
uint256 baseRecovery = originalSlashAmount[msg.sender];
uint256 rateProduct = RECOVERY_RATE_PER_MONTH * monthsElapsed;

// Validate before multiplication to prevent overflow
require(baseRecovery <= type(uint256).max / rateProduct, "Overflow risk");

uint256 maxRecovery = (baseRecovery * rateProduct) / 10000;
```

Alternative simpler approach (if using Solidity 0.8.x):
```solidity
// Solidity 0.8.x provides built-in overflow protection
// Just add bounds checking for reasonableness
uint256 monthsElapsed = (block.timestamp - lock.lastClaimTime) / 30 days;
require(monthsElapsed <= 120, "Max 10 years recovery");  // Cap at 10 years

// The built-in overflow protection will throw on overflow
uint256 maxRecovery = (originalSlashAmount[msg.sender] * RECOVERY_RATE_PER_MONTH * monthsElapsed) / 10000;

// Additional sanity check
require(maxRecovery <= originalSlashAmount[msg.sender], "Recovery exceeds original slash");
```

Choose the simpler Solidity 0.8.x approach with additional bounds checking.

<task type="auto">
  <name>Task 4: Fix checks-effects-interactions in slashLiker()</name>
  <files>contracts/StakingManager.sol</files>
  <action>
Fix potential CEI violation in slashLiker(). The function should validate all conditions before making any state changes.

Current code (lines 98-120) does:
1. Calculate penaltyAmount
2. Update lock.lockedAmount (state mutation)
3. Check lastClaimTime == 0 (check)
4. Record originalSlashAmount (state mutation)
5. Update userReputation (state mutation)
6. Reset hasLiked (state mutation)
7. Emit events (external interaction)

Fix to follow CEI pattern:
```solidity
function slashLiker(address _liker, int256 _penalty, string memory _reason) external onlyOwner nonReentrant {
    // CHECKS: Validate all conditions first
    require(_liker != address(0), "Invalid address");
    require(_penalty < 0, "Penalty must be negative");  // _penalty is negative
    require(bytes(_reason).length > 0, "Reason required");
    require(!hasLiked[_liker] || true, "State check");  // Keep existing logic

    // Calculate penalty amount
    uint256 penaltyAmount = uint256(-_penalty);
    ReputationLock storage lock = reputationLocks[_liker];
    
    // EFFECTS: All state mutations grouped together
    lock.lockedAmount += penaltyAmount;
    if (lock.lastClaimTime == 0) {
        lock.lastClaimTime = block.timestamp;
    }
    originalSlashAmount[_liker] += penaltyAmount;
    userReputation[_liker] += _penalty;
    hasLiked[_liker] = false;
    
    // INTERACTIONS: Events last
    emit ReputationLocked(_liker, penaltyAmount);
    emit AntiSlash(_liker, _penalty, _reason);
}
```

Add input validation at the beginning (require statements) to fail fast before any state changes.
