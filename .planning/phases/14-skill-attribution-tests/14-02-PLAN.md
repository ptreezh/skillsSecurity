## Key Technical Details

### Attribution Key Features
- setStakingManager() is called in fixture (line 41 in fixtures.cjs) - critical for cross-contract calls
- addContribution onlyOwner - reverts for non-owner
- likeSkill uses hasLiked[msg.sender] check (not per skillId)
- addTestReport triggers StakingManager.setPositiveContribution when score > 0

### ContributionType Enum
- GENESIS = 0
- FORK = 1
- AUDIT = 2
- BUGFIX = 3
- TRANSLATION = 4

### Events
- SkillLiked(address indexed user, uint256 indexed skillId) - line 168
- PositiveContributionSet(address indexed user) - from StakingManager

### Data Structures
```solidity
struct Contribution {
    address contributor;
    uint256 share;        // percentage * 100 (e.g., 7000 = 70%)
    ContributionType ctype;
    uint256 timestamp;
    string contentHash;
}

struct TestReport {
    address reporter;
    uint256 severity;    // 1-5
    int256 score;        // Can be negative
    uint256 timestamp;
}

struct Like {
    address user;
    uint256 skillId;
    uint256 timestamp;
}
```

### Mappings
- skillContributions[skillId] -> Contribution[]
- skillTestReports[skillId] -> TestReport[]
- skillLikes[skillId] -> Like[]
- hasLiked[user] -> bool (global, not per skill)

### Like Mechanic Note
The hasLiked mapping is per USER, not per (USER, SKILL). So once a user likes ANY skill, they cannot like any skill again. This is a global like restriction.
