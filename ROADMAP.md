## 🔔 定时检查机制（鞭策系统）

### 每周一自动检查（cron 配置）

```bash
# 每周一 09:00 检查进度
# 使用系统 crontab 或 GitHub Actions

# 检查内容：
1. 读取 ROADMAP.md 当前阶段任务
2. 统计已完成任务数 / 总任务数
3. 计算进度百分比
4. 如果进度 < 80% → 发送警告
5. 如果进度 < 60% → 发送严重警告
6. 如果进度 < 40% → 考虑 Pivot
```

### 检查脚本（check-progress.sh）

```bash
#!/bin/bash
# 检查 ROADMAP 进度

PHASE=$(grep -A 1 "当前阶段" CURRENT_PHASE.txt | tail -1)
TOTAL=$(grep -c "\[ \]" ROADMAP.md)
DONE=$(grep -c "\[x\]" ROADMAP.md)
PROGRESS=$(echo "scale=2; $DONE / $TOTAL * 100" | bc)

echo "当前阶段: $PHASE"
echo "总任务: $TOTAL"
echo "已完成: $DONE"
echo "进度: $PROGRESS%"

if (( $(echo "$PROGRESS < 80" | bc -l) )); then
    echo "⚠️ 警告：进度 < 80%，需要加速！"
    # 发送通知（邮件/Discord/Slack）
fi

if (( $(echo "$PROGRESS < 60" | bc -l) )); then
    echo "🚨 严重警告：进度 < 60%，考虑 Pivot！"
fi

if (( $(echo "$PROGRESS < 40" | bc -l) )); then
    echo "❌ 失败：进度 < 40%，重新评估宪法可行性！"
    exit 1
fi
```

### 手动检查命令（快速）

```bash
# 查看当前阶段进度
grep -A 20 "## Phase $(cat CURRENT_PHASE.txt)" ROADMAP.md | grep -c "\[x\]"
```

## 🎯 鞭策规则（宪法级）

1. **每周一必须检查**，不得跳过
2. **进度 < 80%** → 增加工作时间（每天 +2 小时）
3. **进度 < 60%** → 砍掉非核心功能，聚焦 MVP
4. **进度 < 40%** → 暂停新功能，只修复 Bug 和收集反馈
5. **连续 4 周 < 40%** → 触发宪法修正程序（第七条）
